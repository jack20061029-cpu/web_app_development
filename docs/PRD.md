# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：希望能集中管理與收藏喜歡的食譜，取代散落各處的網頁書籤與手機筆記，讓下廚前備料與實際烹煮時可以更直覺、快速地查閱。
- **目標用戶**：自己（個人使用）
- **核心價值主張**：提供一個輕量、簡單、直覺的食譜紀錄系統。一次呈現好料理所需的所有必要資訊（包含材料、步驟、時間等），並透過分類或標籤功能快速尋找靈感。

## 2. 功能需求

以下為系統核心提供的主要功能列表，並以使用者故事 (User Stories) 呈現：

1. **新增食譜**
   - 作為 **自己 (使用者)**，我希望 **能夠透過介面輕鬆新增一篇食譜，並填寫其基本資訊**，以便 **隨時將新學會或新發現的料理紀錄下來**。
2. **列出材料**
   - 作為 **自己 (使用者)**，我希望 **每份食譜中都能清楚條列出需要的材料與相對應的份量**，以便 **在採買食材與下廚備料時能一目了然**。
3. **列出步驟**
   - 作為 **自己 (使用者)**，我希望 **食譜內部能夠以編號或步驟化方式列出烹飪過程**，以便 **下廚時可以順暢地一步一步跟著操作**。
4. **列出所需時間**
   - 作為 **自己 (使用者)**，我希望 **能夠在食譜標註這道料理預估的準備與烹調時間**，以便 **能夠根據當下有多少剩餘時間，決定要挑戰什麼料理**。
5. **智能分類與標籤管理**
   - 作為 **自己 (使用者)**，我希望 **可以針對食譜加上自訂標籤 (如：低脂、快炒、台式...)，並進行分類管理**，以便 **未來食譜數量增加時，能快速過濾並找到想做的菜色**。

## 3. 非功能需求
- **技術限制**：預計採用 Flask + Jinja2 + SQLite 作為基礎技術選型（若本機開發模式）。
- **效能與安全考量**：
  - 開啟頁面及搜尋過濾時反應需快速不卡頓。
  - 因目標為「個人自用」，無須過於複雜的權限控制機制，考量方便性為主，但不應將本地端資料輕易對外暴露。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**：
  - 食譜的基本 CRUD（新增、讀取/列表與詳情、編輯、刪除）。
  - 食譜需包含欄位：名稱、所需時間、材料表、步驟清單。
- **Should Have (應該有)**：
  - 基本的標籤新增與基於標籤的資料過濾/搜尋功能。
  - 響應式介面 (RWD)，確保在手機或平板上看時排版不會跑版，方便一邊看著手機一邊下廚。
- **Nice to Have (加分項)**：
  - 食譜封面圖片的上傳與顯示。
  - 智能輸入/自動分析從外部複製貼上的材料文字轉換成列表格式。
  - 可以將所需的材料一鍵加入「購物買菜清單」。
